## 三、技术架构

### 3.1 整体架构

- **单文件 SPA**：所有页面（首页/统计/资金/报销/设置）通过 `switchPage()` 切换 `display` 属性实现
- **数据存储**：双引擎持久化——`localStorage`（主存储）+ `IndexedDB`（冗余备份）
- **离线能力**：Service Worker 缓存关键资源，支持离线使用
- **PWA 特性**：可添加到主屏幕、Web Share Target API（接收其他App分享的截图）

### 3.2 技术栈

| 层次 | 技术 |
|------|------|
| 前端框架 | 无框架，原生 JavaScript |
| 样式 | 纯 CSS（CSS 变量 + 渐变 + Safe Area 适配） |
| 数据存储 | localStorage + IndexedDB |
| 离线缓存 | Service Worker（Cache API） |
| 图片处理 | Canvas API + FileReader |
| OCR 识别 | 基于正则表达式的文字匹配 + Tesseract.js CDN（预留） |

### 3.3 数据流

```
用户操作 → app.js 处理 → 更新内存状态
                    → localStorage 持久化
                    → IndexedDB 冗余备份
                    → 自动备份到 localStorage
                    → 5分钟定时自动备份
```
